Topic3: Pipes and change detection

Angulars' change detection process runs after every DOM event: every keystroke, mouse move, timer tick, and server response. This could be expensive in case of pipes

So Angular picks a simpler, faster change detection algorithm when you use a pipe.

If we add an element in the array we are iterating, like this heroes.push(another hero)

This would work fine without filter, but if used with filter this is not going to update the view, because same array, no change, no display update

If you mutate the array, no pipe is invoked and the display isn't updated; if you replace the array, the pipe executes and the display is updated.

Topic3: Pure and impure pipes:

1: Pure pipes

Angular executes a pure pipe only when it detects a pure change to the input value. A pure change is either a change to a primitive input value (String, Number, Boolean, Symbol) or a changed object reference (Date, Array, Function, Object).

2: Impure pipes

Angular executes an impure pipe during every component change detection cycle. An impure pipe is called often, as often as every keystroke or mouse-move.
